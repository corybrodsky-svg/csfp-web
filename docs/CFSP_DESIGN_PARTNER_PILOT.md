# CFSP Design Partner Pilot

## Pilot Purpose
Run a focused 30-60 day private pilot with one healthcare simulation program to validate whether CFSP improves the operational workflow around SP staffing, communications, training readiness, materials, and day-of coordination.

The pilot should be scoped around real workflow patterns, but it must not include PHI, student grades, real patient data, or sensitive assessment outcomes.

## Suggested Pilot Scope
- One simulation program, center, or course team.
- Two to five simulation events or event series.
- A small operator group: simulation director, SP program lead, Sim Ops coordinator, and one backup admin.
- A small SP group only if the organization is comfortable using the SP portal in a controlled way.
- Hybrid communication workflow: portal where appropriate, manual/email/Microsoft Forms where needed.

## What CFSP Will Support
- Event command center setup for selected pilot events.
- SP coverage tracking and assignment visibility.
- SP shift offers and response capture.
- SP attendance status tracking for day-of operations.
- SP communication preference tracking.
- Portal invite onboarding for selected SPs, if the pilot team wants to test it.
- Demo/operator checklists and smoke-test discipline before important walkthroughs.
- Pilot feedback capture after each demo, pilot call, or event review.

## What Remains Manual
- Sending emails, texts, and Microsoft Forms.
- Microsoft Graph or Outlook calendar integration.
- Institutional SSO, provisioning, and access-policy review.
- Data import cleanup from legacy spreadsheets.
- Final assignment decisions and payroll processes.
- Security review documentation beyond the current pilot materials.
- Any workflow that involves PHI, real patient data, student grades, or assessment outcomes.

## Success Criteria
- Operators can find event staffing status faster than in the current workflow.
- SP availability and responses are easier to review against event needs.
- Communication coverage is clearer across portal, email, Microsoft Forms, manual, and do-not-contact workflows.
- Training/material readiness is easier to explain during event prep.
- Day-of attendance status is easier to track and share with the operations team.
- SP portal users see only their own SP-facing information.
- The pilot team can name which workflows should stay, change, or remain manual before paid rollout.

## Roles And Responsibilities
CFSP:
- Provide the demo environment, pilot setup guidance, and documentation.
- Support agreed pilot workflows and troubleshoot issues.
- Keep pilot scope conservative and privacy-safe.
- Record feedback and convert it into a practical roadmap.

Pilot organization:
- Identify the pilot owner and operator users.
- Select pilot events that do not require PHI or student grades in CFSP.
- Provide fake/demo data for demos and approved non-sensitive operational data for pilot use.
- Decide which SPs, if any, should test the portal.
- Keep real invite links, SP contact details, and private staffing notes protected.

## Privacy And Safety Assumptions
- No PHI.
- No student grades, IDs, assessment outcomes, or remediation records.
- No real patient data or case details that identify a patient.
- SP portal users must only see their own shifts, responses, upcoming items, and attendance status.
- Invite links are sensitive and should be handled like account access links.
- Raw invite URLs appear only once when created.
- Admin-only notes, emails, phone numbers, roster data, invite history, token hashes, and other SP data must not be exposed to SP users.

## Suggested Timeline
Week 0:
- Confirm pilot scope, data boundaries, users, and selected events.
- Run demo using fake data.

Weeks 1-2:
- Configure organization settings and pilot events.
- Review SP communication preferences and selected workflow paths.
- Decide whether portal invite onboarding will be tested.

Weeks 3-5:
- Use CFSP in parallel with existing workflow for selected events.
- Capture feedback after each event or weekly pilot check-in.

Weeks 6-8:
- Review success criteria.
- Identify required fixes, manual workflows, and license-fit concerns.
- Decide whether to move toward a paid design partner annual license.
